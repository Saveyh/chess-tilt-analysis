# Publication Notes

This folder documents the changes made to prepare the project for a public GitHub repository.

## What was intentionally kept out of the public version

- The original PDF report, because it contains author names, student numbers, a university declaration, and author metadata in the PDF file itself.
- The raw CSV export, because it contains public chess usernames and was not normalized for publication.
- The original course script, because it contains a local Windows path, package installation instructions inside the script, and file references that do not match the files in this folder.

## What was added for the public version

- A repaired and anonymized dataset generated by `scripts/prepare_public_dataset.py`.
- A cleaned R analysis script in `analysis/chess_tilt_analysis.R`.
- A generated markdown summary in `docs/public_summary.md`.
- A `.gitignore` that prevents the private course files from being committed by mistake.

## Recommended GitHub presentation

- Use the repository exactly as a portfolio copy of a course project, not as the original graded submission.
- Keep the README focused on the question, the dataset, the methodology, and the key findings.
- If you later add more school projects, use the same publication standard: anonymized data, reproducible scripts, and no institutional or personal administrative documents.
